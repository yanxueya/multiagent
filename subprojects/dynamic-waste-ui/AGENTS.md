# dynamic-waste-ui 开发约束

本目录只负责用户界面、监控面板、人工复核和可视化，不承载模型训练、知识图谱核心推理或 ROS2 硬件执行逻辑。

## 设计原则

- 优先做工作台式界面：信息密集、状态清晰、操作可确认。
- 主界面必须体现多智能体信息流，而不是做成营销式首页。
- 高风险机器人动作必须有明确确认状态。
- 人工复核、unknown 状态、VLM 冲突和 ROS2 命令预览必须在交互上分清。
- 不使用装饰性过强的渐变、光效或大面积宣传图作为主界面。

## 工程边界

- 不嵌入模型训练逻辑。
- 不把实验论文文档放入前端目录。
- 不直接连接机械臂硬件；前端只通过后端 API 或 agent bridge 读写结构化状态。
- 不把 LLM 自由文本直接发送给 ROS2；ROS2 只接收受 schema 约束的结构化命令。

## 与其他子项目关系

- `dynamic-waste-agent` 提供 LangGraph 多智能体状态、规划结果和人工复核请求。
- `dynamic-waste-kg` 提供 KG 状态、类别先验、实例记忆、unknown 记忆和事件日志。
- `dynamic-waste-ros2` 后续提供机器人执行状态和结构化动作接口。
- `dynamic-waste-sim` 后续提供仿真或离线回放数据。
