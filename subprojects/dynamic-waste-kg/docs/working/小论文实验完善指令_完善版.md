# 建筑废弃物 YOLO-VLM-动态知识图谱小论文：实验完善指令

## 1. 研究定位

目标期刊为 *Automation in Construction*。本文是一篇受控环境下的自动化方法与原型验证论文，研究对象为 12 类建筑废弃物的二维实例分割、受约束视觉语言模型（VLM）复核，以及分层动态知识状态构建。

本文唯一核心主张为：

> 本研究将建筑废弃物实例分割结果转换为包含类别语义、安全处理约束、状态版本与事件溯源的二维任务策略就绪状态，为后续三维定位和机器人规划提供结构化接口。

不得主张已完成真实机械臂抓取、RealSense 三维定位、空间关系推断、LangGraph 多智能体协商、ROS2 执行、施工现场部署或完全自主决策。LLM 自报置信度不得作为视觉准确率证据。

## 2. 论文逻辑与贡献

### 2.1 问题

YOLO 实例分割输出类别、置信度和掩膜，但不能直接回答对象是否可自动处理、是否应人工复核，以及判断的来源和状态变化过程。传统类别知识图谱也常混合稳定知识、当前对象和过程事件，难以为后续规划提供可审计状态。

### 2.2 方法回应

系统由三层构成：

1. 长期类别层：材料类别语义和保守处理先验；
2. 短期实例记忆层：当前对象的二维视觉与策略状态；
3. 事件日志层：观测、复核、更新与路由的不可变记录。

YOLO11n-seg 是感知底座；VLM 仅处理低置信度或策略敏感对象；知识状态模块将最终类别投影为可查询的处理路由。

### 2.3 规范贡献表述

1. 提出面向建筑废弃物分拣任务准备的分层动态知识状态模型，分离长期类别先验、短期对象实例与不可变事件日志。
2. 建立 YOLO11n-seg 到知识状态的结构化接口，将类别、置信度、边界框和掩膜转化为带有版本和来源的实例状态，并投影保守处理策略。
3. 提出受类别白名单、JSON schema、风险规则和人工升级机制约束的 VLM 复核策略，避免开放式 LLM 分类风险。
4. 在 12 类数据集、批量测试案例和受控动态状态案例上验证感知性能、复核行为、安全路由与事件可追溯性。

训练模型、数据集和单图原型均是验证条件，不是与上述方法并列的创新。

## 3. 实现边界

长期层保留：

```text
class_name, risk_level, fragility, nominal_graspability,
pollution_prior, default_handling_mode, needs_llm_review,
auto_processable, nominal_grasp_difficulty, provenance
```

`recognition_difficulty` 不应作为材料固有知识；它依赖模型、相机和数据集，应由类别 AP、混淆率等模型性能统计给出。

短期层在本文仅激活：

```text
instance_id, class_name, yolo_confidence, final_class,
final_confidence, bbox_2d, mask_polygon, review_status,
route_decision, task_status, timestamp, state_version, last_action
```

`center_xyz`、`bbox_3d`、`blocked_by`、`supports`、`grasp_candidates` 和 `safe_grasp_score` 仅作为后续 RGB-D 和机器人研究接口。

必须定义以下更新操作并写入事件日志：

```text
CreateInstance(observation)
UpdateObservation(instance, observation)
ApplyVLMReview(instance, review)
ProjectCategoryPolicy(instance, category)
RouteTaskState(instance)
AppendEvent(event)
```

每次更新增加 `state_version`。快照必须能够由同一 `instance_id` 的事件链解释。

## 4. 实验任务

### E0：数据集审计与可复现基线

- 统计 train/val/test 的图像数、对象实例数、每类实例数；不要把标注文件数写成对象实例数。
- 检查同一原图及其增强版本是否跨 train/val/test，防止数据泄漏。
- 固定 `data.yaml`、类别映射、随机种子、权重、训练命令、环境版本。
- 输出每类 AP、每类 Precision/Recall、训练曲线、混淆矩阵和单图推理时间。
- 将 YOLO11n-seg 表述为适合 8 GB 显存快速迭代的轻量原型，不声称其为最优检测器。

建议输出：

```text
artifacts/e0_dataset_audit/class_distribution.csv
artifacts/e0_dataset_audit/split_leakage_report.md
artifacts/e0_dataset_audit/model_environment.json
artifacts/e0_dataset_audit/per_class_metrics.csv
```

### E1：实例分割性能与错误分析

使用独立 test 集报告总体 Precision、Recall、Box/Mask mAP50、Box/Mask mAP50-95，以及类别级结果。

从归一化混淆矩阵中选择非对角线数值最大的 2–4 个类别对，不要预先认定 `gypsum_board`、`foam`、`hard_plastic` 或 `glass` 一定最易混淆。每个类别对报告真实类别、预测类别、混淆数量、该真实类的混淆率及代表图像 ID。

制作四类定性案例图：

| 类型 | 选择规则 | 解释目的 |
|---|---|---|
| 典型成功 | 类别正确且 mask IoU 高 | 感知底座在清晰对象上的能力 |
| 遮挡失败 | 重度遮挡导致漏检、低置信度或分割不全 | 复杂堆叠边界 |
| 边界失败 | 类别正确但 mask 粘连、断裂或偏移 | 对后续几何/抓取接口的影响 |
| 材质混淆 | 目标匹配而类别错误 | VLM 复核与人工升级的适用条件 |

这些图只用于解释错误机制，不能替代定量结果。

### E2：受约束 VLM 复核

VLM 必须接收视觉输入：原图、bbox 裁剪图、mask-overlay 图、12 类白名单及严格 JSON schema。若仅收到 YOLO 标签文本，则只能称为“语义规则审查”，不能称为视觉复核。

在 val 集选择并固定阈值 `tau`：

```text
trigger = (yolo_confidence < tau)
          OR (candidate class is policy-sensitive)
```

至少将 `glass`、`gypsum_board` 和 `asbestos_suspect` 列为策略敏感类；其他类别由 val 集混淆分析决定。不得用 test 集调整 `tau`、类别名单或提示词。

比较组：

```text
C1: YOLO-only
C2: YOLO + constrained VLM review
C3: YOLO + constrained VLM review + dynamic knowledge-state routing
```

固定模型名称/版本、prompt、temperature（推荐 0）、max_tokens、输入图格式、JSON schema 和回退规则。推荐输出：

```json
{
  "decision": "agree | change | uncertain",
  "proposed_class": "one of 12 allowed classes",
  "requires_human_review": true,
  "reason": "short visual rationale"
}
```

必须报告：

```text
review_coverage
reviewed_subset_accuracy_before
reviewed_subset_accuracy_after
correction_rate
harmful_override_rate
invalid_response_fallback_rate
human_escalation_rate
mean_vlm_latency
```

`harmful_override_rate` 是 YOLO 原类别正确、VLM 却改为错误类别的比例。VLM 自报置信度只可记录，不可作为准确率依据。对高风险或不确定对象，优先升级人工复核，而不是自动改写类别。

若 VLM 返回非法类别、格式错误、`uncertain` 或与安全规则冲突：

```text
retain YOLO raw record
review_status = human_review
route_decision = HUMAN_REVIEW_REQUIRED
```

### E3：知识状态安全路由

为每个与真实标注匹配的测试实例，以其**真实类别**查询长期策略表，构造预期路由；绝不能用系统预测类别构造评价真值。

路由标签为：

```text
AUTO_CANDIDATE
SUPERVISED_CANDIDATE
HUMAN_REVIEW_REQUIRED
```

最小规则：

```text
auto_processable=true 且非敏感且满足置信度条件 -> AUTO_CANDIDATE
robot_with_supervision 或已触发复核 -> SUPERVISED_CANDIDATE
human_review / human_only / VLM uncertain / 非法输出 -> HUMAN_REVIEW_REQUIRED
```

报告：

```text
Unsafe Automation Rate
Restriction Recall
Over-conservative Rate
Policy Consistency Rate
```

其中，真实应受限实例由真实类别对应的 `auto_processable=false` 或 `handling_mode in {human_review, human_only}` 决定。

长期策略库应称为“保守原型策略库”。类别背景与石棉安全边界可引用规范或权威材料；默认抓取和处理方式属于工程先验。可选增强项是邀请 3–5 名相关人员审阅风险等级、默认处理方式及人工升级规则。

### E4：动态状态更新

最低要求为 30 个实例级受控事件回放案例，每条案例 3–4 个状态：

```text
t1: YOLO initial observation
t2: VLM review or repeated observation
t3: class/confidence/task status update
t4: policy re-routing and event append
```

该验证只能命名为 `controlled event-replay validation`。

推荐同步采集真实短序列：固定 RGB 相机拍摄 20–30 组真实废弃物场景，每组 3–10 帧。通过人工新增、移除、遮挡、重新摆放或确认对象生成状态变化。重点覆盖策略敏感类和实际混淆类。

在固定相机、短时相邻帧和有限位移前提下，可用：

```text
same class + mask IoU >= 0.5 in adjacent frames
-> inherit instance_id
otherwise -> create a new instance_id
```

必须说明这是工程性原型假设，不是本文创新。

必报指标：

```text
instance_update_success_rate
event_chain_completeness
state_version_consistency
temporal_policy_consistency
```

被拿走的对象应显式更新为 `task_status = removed`，而不是在最新快照中静默消失。

## 5. 图谱能力证明

论文必须展示三类可复现查询：

```text
Q1: 当前哪些实例可作为自动处理候选？
Q2: 哪些高风险或不确定实例必须人工复核？原因是什么？
Q3: 指定 instance_id 从观测到最终路由经历了哪些事件？
```

这些查询是图谱区别于静态 JSON 导出的关键证据。

## 6. 图表与写作检查

建议图：总体框架、三层状态模式、VLM 状态机、训练结果、四类 E1 案例、连续状态更新案例、图谱查询案例。

建议表：数据集与类分布、长期策略库及来源、训练环境、类别级性能、VLM 对照、安全路由、失败模式。

投稿前确认：

- [ ] 阈值、prompt、白名单均在 val 集冻结；
- [ ] test 集只用于最终报告；
- [ ] VLM 确实输入图像，或已正确降级命名；
- [ ] 动态性由连续状态或受控事件回放验证；
- [ ] 长期先验与实例级实时状态分离；
- [ ] 识别难度未被误写为材料固有属性；
- [ ] 报告 VLM 错误改写、回退和人工升级，不只报告成功案例；
- [ ] RGB-D、三维关系、抓取、ROS2 和多智能体均被明确写为后续工作。

## 7. 可接受的结论边界

> The proposed framework supports the conversion of instance-segmentation outputs into policy-aware and traceable two-dimensional scene states under controlled evaluation settings. It does not yet validate three-dimensional geometric reasoning, physical grasp execution, or autonomous multi-agent coordination.

禁止使用 “fully autonomous”, “field-ready”, “safe grasping achieved” 或 “multi-agent decision-making validated” 等超出证据范围的表述。
