# scripts

本目录只放命令行入口，不放核心业务逻辑。可复用逻辑应放在 `wastekg/` 的领域子包中，脚本只负责解析参数、调用库函数和写出结果。

## 子目录

- `data/`：数据集审计、合并、冻结和分组入口。
- `yolo/`：YOLO 分割训练、评估和样本筛选入口。
- `llm/`：大模型配置检查和轻量验证入口。
- `rgbd/`：RealSense/RGB-D 单帧采集、分组数据采集和图谱接入入口。
- `graph/`：知识图谱导出、Neo4j 导入和图谱预测入口。
- `paper/`：论文实验所需的样本导出入口。
- `tools/`：临时但可复用的小工具。

## 原则

- 不在脚本中堆业务逻辑。
- 不在脚本中硬编码个人绝对路径。
- 训练脚本只提供入口；涉及模型训练时，由用户在本机确认显存后手动运行。
- 新脚本应放入最接近职责的子目录，而不是直接放在 `scripts/` 根目录。
